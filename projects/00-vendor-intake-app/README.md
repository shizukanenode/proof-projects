# 00 Vendor Intake App

## Objective
Simulate a basic vendor security intake workflow by collecting questionnaire responses, applying a simple risk rating, and documenting the review.

## Environment
- Project type: local Python and Markdown project
- Main script: `app.py`
- Supporting files: Markdown templates and review notes

## Tasks Completed
- Created a basic vendor security questionnaire
- Defined simple risk scoring guidance
- Recorded a sample vendor review
- Built a small Python script to display vendor review information

## Skills Demonstrated
- Vendor security review workflow
- Third-party risk assessment basics
- Security documentation
- Python scripting
- Structured review process

## Files in This Project
- `questionnaire_template.md` - vendor security review questions
- `risk_scoring.md` - simple risk rating guidance
- `sample_vendor_review.md` - example completed vendor review
- `app.py` - small Python script that displays vendor review information

## Key Takeaways
This project shows a simple third-party security review process: gather vendor details, review controls, assign a risk level, and document the result clearly.

## Next Improvements
- Add weighted scoring instead of simple guidance
- Export review results to CSV or JSON
- Track multiple vendors instead of one sample review
- Add approval status and remediation notes
